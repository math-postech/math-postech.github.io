我们的网站原来只发 colloquium 之类的学术活动信息，现在要加上课程教学信息发布。

问题来了：顶层目录叫什么？考虑了三个名字——`teaching/` 太随意，`curriculum/` 其实是"培养方案"的意思不是具体的课，最后选了 `courses/`，精确地表达"这里放的是一门一门的课"。

目录结构是 `courses/MAT203/2026-spring/readme.md`，课号做中间一层、学期做最内层，和已有的 `colloquium/2026-spring/` 保持风格统一。

如果反过来用 `curriculum/`，以后别人看到会以为是整个专业的课程设置，而不是某门课的作业和讲义，会造成混淆。
