Tutorial: Week 6 — Projection Structure ​

For: Teaching Assistants conducting tutorial sessions (Apr 2, 2026) Related Lectures: Lecture 7: Linear Transformation and Projection, Lecture 8: Cross-Filling ProjectionsDuration: 50 minutes Prerequisites: Projection definition (P2=P), cross-filling P=UV with VU=I, rank(P)=trace(P), rank-1 projection characterization (Corollary 4.1), rank subadditivity rank(A+B)≤rank(A)+rank(B)

Problem 1: When Is a Sum of Projections a Projection? ​

Let P1,P2 be projections (P12=P1, P22=P2).

(a) Find the condition on P1,P2 so that P1+P2 is also a projection.

Hint: Expand (P1+P2)2=P1+P2 and simplify using Pi2=Pi.

Solution

(P1+P2)2=P1+P2 expands to:

P12+P1P2+P2P1+P22=P1+P2

Since Pi2=Pi, this simplifies to:

P1P2+P2P1=0

Claim: This forces P1P2=P2P1=0.

Proof (Note: This argument requires 2≠0, i.e., we work over R or any field of characteristic ≠2.)

Right-multiply the equation P1P2+P2P1=0 by P1:

(1)P1P2P1+P2P1=0⟹P1P2P1=−P2P1

Left-multiply the original equation by P1:

(2)P1P2+P1P2P1=0⟹P1P2P1=−P1P2

From (1) and (2): −P2P1=−P1P2, so P1P2=P2P1.

Substituting back into P1P2+P2P1=0:

2P1P2=0⟹P1P2=0

Therefore P1P2=P2P1=0.

(b) Verify the converse: if P1P2=P2P1=0, then P1+P2 is a projection.

Solution

(P1+P2)2=P12+P1P2+P2P1+P22=P1+0+0+P2=P1+P2. ✓

(c) Give a pair of projections P1,P2 such that P1+P2 is not a projection.

Solution

Take P1=P2=(1000). Then P1+P2=(2000), and (P1+P2)2=(4000)≠P1+P2.

(Here P1P2=P1≠0, so the condition fails.)

Remark

Connection to Lecture 9: The condition P1P2=0 (with both being projections) is exactly the definition of a compatible family {P1,P2}. This problem shows: P1+P2 is a projection if and only if {P1,P2} is a compatible family.

Problem 2: The Rank-1 Projection Characterization ​

(a) If a matrix A has rank(A)=1 and trace(A)=1, prove that A is a projection.

Proof

Since rank(A)=1, write A=uvT for some column vectors u,v.

Then trace(A)=trace(uvT)=vTu=1.

Compute:

A2=(uvT)(uvT)=u(vTu)vT=u⋅1⋅vT=A◻

(b) Does this generalize? Give an example of a matrix with rank(A)=2 and trace(A)=2 that is not a projection.

SolutionA=(1101)

rank(A)=2, trace(A)=1+1=2.

But A2=(1201)≠A, so A is not a projection.

Remark

Why Does Rank 1 Work But Rank 2 Doesn't?

For rank-1 matrices, the entire structure is captured by the single number vTu: A2=(vTu)⋅A. So trace=1 directly forces A2=A.

For rank ≥2, the product form A=UV gives A2=U(VU)V, and VU is a matrix (not a scalar). Knowing trace(VU)=2 does not force VU=I2. In the example above, VU=A itself, which has trace=2 but VU≠I2.

This is what makes Lecture 8's theorem (P2=P⇒VU=I) so remarkable: the projection property, not the trace, is what forces VU=I.

Problem 3 (Challenge): A Characterization of Projections by Rank ​

Prove: rank(I−P)+rank(P)=n if and only if P2=P.

Note: This problem may be deferred to a future tutorial depending on time.

(a) (⇐): If P2=P, show that rank(I−P)+rank(P)=n.

Hint: Use rank=trace for projections.

Proof

If P2=P, then I−P is also a projection: (I−P)2=I−2P+P2=I−P.

By rank = trace (Lecture 8, Theorem 3.1):

rank(I−P)+rank(P)=trace(I−P)+trace(P)=trace(I)=n◻

(b) (⇒): This is the harder direction. If rank(I−P)+rank(P)=n, show that P2=P.

Hint: It suffices to show P(I−P)=0. Use the block matrix rank technique: embed P and I−P into a block matrix, apply rank subadditivity, then use the cross-filling philosophy (cf. Lecture 9 slides) to conclude the off-diagonal block vanishes.

Proof

Goal: Show P(I−P)=0, which is equivalent to P=P2.

Step 1: Embed into block matrices.

Observe that:

rank(P0P0)=rank(P),rank(I−PI−P00)=rank(I−P)

(Each block matrix has the same column space, respectively row space, as its nonzero block.)

Step 2: Apply rank subadditivity.

Their sum is:

(P0P0)+(I−PI−P00)=(II−PP0)

By rank(A+B)≤rank(A)+rank(B):

rank(II−PP0)≤rank(P)+rank(I−P)=n

Step 3: Use the cross-filling lower bound.

The block matrix (II−PP0) contains the n×n identity I in its top-left corner. Cross-filling I into n rank-1 pieces, each piece is supported on a single row and column of the top-left block — none of these pieces touch the bottom-right block. Therefore:

rank(II−PP0)≥n

(The n rank-1 pieces from I alone contribute rank n.)

Step 4: Combine and conclude.

From Steps 2 and 3:

rank(II−PP0)=n

Now, the general principle for block matrices of the form (IABC):

Cross-filling the identity I produces n rank-1 pieces. After subtracting these from the full matrix, the remainder is concentrated in the bottom-right block: C−BA. Therefore:

rank(IABC)=n+rank(C−BA)

Applying this with A=I−P, B=P, C=0:

n=n+rank(0−P(I−P))

So rank(P(I−P))=0, which means P(I−P)=0, i.e., P=P2. ◻

Remark

What This Tells Us

The condition "rank(I−P)+rank(P)=n" says that P and I−P together account for all n dimensions with no overlap and no waste. This is exactly the geometric content of being a projection: the space splits cleanly into the "floor" Col(P) and the "sunlight direction" Null(P)=Col(I−P).

For non-projections, rank(I−P)+rank(P)>n — dimensions are "double-counted."

TA Notes ​

Time allocation ​

ProblemSuggested TimePriority
Problem 1 (sum of projections)20 minCore — the main result of this tutorial
Problem 2 (rank-1 characterization + counterexample)15 minCore — (a) is a warm-up proof, (b) shows the boundary
Problem 3 (rank characterization)15 minChallenge — defer if short on time

Suggested flow ​

  1. Start with Problem 2 as a warm-up — it reviews Lecture 8's Corollary 4.1 and the counterexample in (b) motivates "what makes projections special."
  2. Problem 1 is the main result: students discover the compatible family condition. Let them expand (P1+P2)2 themselves before showing the right/left multiply trick.
  3. Problem 3 only if time permits. The (⇐) direction is a quick application of rank = trace. The (⇒) direction uses block matrix rank techniques from Lecture 9 slides and can be presented as a "preview" of the cross-filling philosophy applied to block matrices.

Common mistakes ​

  • Problem 1: Students may stop at P1P2+P2P1=0 and not realize this forces P1P2=0. The right/left multiply trick is the key insight. Emphasize that we need char≠2.
  • Problem 2(a): Students may try to verify A2=A by brute force instead of using the vTu=1 shortcut. Remind them: for rank-1 matrices, everything reduces to a single scalar.
  • Problem 2(b): The simplest counterexample is I+N where N is nilpotent (e.g., (1101)). If students struggle, suggest: "find a matrix with all diagonal entries 1 but A2≠A."
  • Problem 3: The block rank equality rank(IABC)=n+rank(C−BA) is the key tool. Students may accept it on faith for now; it will be fully justified in later material on block elimination.
